# 公安指挥任务控制台 UI 参考

## 概览

当前方案采用“卡片工作区 + 可左右吸附智能协同侧栏”的布局。页面用于公安指挥大厅或警员工作台，视觉定位是高密度指挥工具：稳定、可扫读、少解释文案，不做营销页或炫技大屏。

核心目标：

- 业务卡片工作区只展示任务执行后产生的业务结果。
- 智能协同侧栏可吸附左侧或右侧，上半区为系统回执和任务消息，下半区为流程列表。
- 侧栏可收起为悬浮“智能协同”入口，入口沿当前吸附边缘上下拖动，也可拖拽到另一侧边缘吸附，点击展开。
- 页面固定为浏览器视口高度，不让 body 或底部无限延伸。
- 所有滚动区域都可滚动，但不显示滚动条。

## 版本边界

第一版智能协同消息区展示 runtime 生成的系统回执和任务消息，不展示主 agent 自由对话。第二版接入主 agent 后，智能协同消息区可以同时展示主 agent 消息和 runtime 任务消息，但必须保留消息归属。

## 组件

- 顶部标题栏：只展示产品名称“公安指挥任务控制台”。
- 卡片工作区：视频、地图、表格、文本、JSON 等业务卡片。
- 卡片操作：每张卡片右上角只保留图形按钮，`▴ / ▾` 控制收起展开，`□ / ×` 控制最大化恢复。
- 智能协同：系统回执、任务消息和底部输入区。
- 输入区：输入框单行起步，右侧图形按钮独立固定高度；输入多行时不撑高按钮。
- 预案指引：流程阶段和子动作列表。
- 流程状态：视觉上通过子项前置圆形符号体现，不在每行末尾显示“已完成/执行中/等待”等文字；每个状态符号必须有可访问语义。

## 侧栏收起态

```text
┌──────────────────────────────────────────────────────────────────────────────┐
│ [警] 公安指挥任务控制台                                                       │
├──────────────────────────────────────────────────────────────────────────────┤
│ 卡片工作区                                                        智能协同    │
│                                                                  悬浮入口    │
│                                                                  可上下拖动  │
└──────────────────────────────────────────────────────────────────────────────┘
```

收起态行为：

- 侧栏宽度变为 `0px`，不占用卡片区域宽度。
- 悬浮入口只显示竖排“智能协同”文字。
- 悬浮入口可沿当前吸附边缘上下拖动。
- 悬浮入口可拖拽到另一侧边缘并吸附。
- 点击文字展开当前吸附侧的侧栏。
- 拖动时不触发展开。

## 卡片行为

- 卡片在对应任务有结果后出现，不提前占位等待状态。
- 卡片右上角不展示任务状态 badge。
- 卡片默认高度统一为 `360px`。
- 卡片工作区按自身可用宽度和高度自适应布局。
- 桌面布局优先保持 3 列卡片。
- 可用宽度不足以保持 3 列时降为 2 列或 1 列。
- 卡片列数由卡片工作区当前宽度决定，不由全 viewport 宽度直接决定。
- 卡片内部内容可滚动或横向适配，但滚动条不显示。
- 收起卡片只保留标题栏。
- 最大化卡片覆盖卡片工作区，不覆盖智能协同侧栏。

## 智能协同侧栏

智能协同侧栏由上下两个等高区域组成：

- `智能协同`：上半区，显示系统回执和任务消息。
- `预案指引`：下半区，显示流程阶段和动作。

视觉规则：

- 侧栏整体背景与卡片区不同。
- 两个区域之间保留 `8px` 间隔。
- 两个区域各自有边框、圆角和略深底色，明确表现为同一侧栏里的上下两块。
- 两个区域都独立滚动，滚动条不显示。

## 输入区

- 输入区固定在智能协同区域底部。
- 输入框单行起步，高度为 `38px`。
- 输入框可以增长到多行，但右侧按钮保持 `38px × 38px`，不被 textarea 撑高。
- 运行中按钮显示停止图形 `■`。
- 未运行时按钮显示发送图形按钮。

## 流程状态

流程子项不使用文字状态列。视觉状态通过每个子项前面的圆形符号表达：

- 已完成：圆圈内 `√`。
- 执行中：loading 旋转圆圈。
- 失败：圆圈内 `×`。
- 等待：空圆。
- 已停止：圆圈内 `!`。

每个状态符号必须通过 `aria-label`、`title` 或同等语义文本暴露状态含义，不能只依赖颜色或图形本身。

选中的流程子项使用行背景和左侧色条提示。

## 滚动

- 页面主体固定在 `100dvh`。
- 卡片工作区独立滚动。
- 消息区独立滚动。
- 预案指引独立滚动。
- 表格、JSON 等卡片内部可滚动。
- 所有滚动区域隐藏滚动条，但保留滚轮、触控板和触摸滚动能力。

## 可访问性

- 悬浮入口、收起、最大化按钮保留 `aria-label`。
- 系统回执区域使用 `aria-live="polite"`。
- 按钮保留可见 focus ring。
- 状态不能只依赖颜色，流程状态符号需保留语义映射。
